# Support

Thanks for using WorldSignal! This page explains where to get help.

## Documentation first

Most questions are answered in the docs:

- [Architecture](docs/ARCHITECTURE.md) — how the system fits together.
- [API](docs/API.md) — GraphQL and REST surface.
- [Database](docs/DATABASE.md) — schema and data model.
- [Configuration](docs/CONFIGURATION.md) — environment variables and LLM keys.
- [Deployment](docs/DEPLOYMENT.md) — running WorldSignal in production.
- [Runbook](docs/RUNBOOK.md) — operational procedures.

If you are setting up a development environment, see
[CONTRIBUTING.md](CONTRIBUTING.md).

## Questions, ideas, and discussion

Use [GitHub Discussions](https://github.com/jigardafda/WorldSignal/discussions)
for:

- How-to and usage questions ("Q&A").
- Sharing and proposing ideas.
- General conversation about the project.

## Bug reports

Open a [GitHub issue](https://github.com/jigardafda/WorldSignal/issues) **only
for reproducible bugs and confirmed defects**. Please use the bug report form
and include version/commit, environment, and reproduction steps.

For feature ideas, prefer the feature request form or a Discussion.

## Security issues

Do not file public issues for security problems. Follow
[SECURITY.md](SECURITY.md) to report vulnerabilities privately.

## Code of Conduct

All support interactions are governed by our
[Code of Conduct](CODE_OF_CONDUCT.md).
